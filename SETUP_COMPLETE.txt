🚀 COMO COMEÇAR

### OPÇÃO 1: Local (Recomendado para Desenvolvimento)

1. Ativar ambiente virtual:
   source venv/bin/activate

2. Instalar dependências:
   pip install -r requirements.txt

3. Executar aplicação:
   streamlit run hydrosim_rf.py

OU usar o script automatizado:
   ./start.sh

OU usar Makefile:
   make run

Acesso: http://localhost:8501

### OPÇÃO 2: Docker (Recomendado para Produção)

1. Iniciar container:
   docker-compose up -d

OU com Makefile:
   make docker-up

2. Ver logs:
   docker-compose logs -f

Acesso: http://localhost:8501

📋 COMANDOS ÚTEIS

Verificar instalação:
  python3 check_installation.py

Com Makefile:
  make help        # Ver todos os comandos
  make install     # Instalar dependências
  make run         # Executar localmente
  make docker-up   # Iniciar Docker
  make docker-down # Parar Docker
  make clean       # Limpar arquivos temporários

Com Docker:
  docker-compose up -d          # Iniciar background
  docker-compose logs -f        # Ver logs
  docker-compose down           # Parar
  docker-compose restart        # Reiniciar

🔍 PRÓXIMOS PASSOS

1. Verificar instalação:
   python3 check_installation.py

2. Preparar dados:
   - Ter um DEM em GeoTIFF
   - Opcionalmente: shapefiles com polígonos

3. Copiar arquivos para data/input/:
   cp seu_dem.tif data/input/

4. Iniciar aplicação:
   make run        # Local
   make docker-up  # Docker

5. Acessar em: http://localhost:8501

🆘 SOLUÇÃO DE PROBLEMAS

Porta 8501 já em uso?
  streamlit run hydrosim_rf.py --server.port=8502

GDAL não encontrado?
  # Linux: sudo apt-get install gdal-bin libgdal-dev libproj-dev
  # macOS: brew install gdal proj geos
  # Use Docker se continuar com problemas

Módulo não encontrado?
  source venv/bin/activate
  pip install -r requirements.txt --upgrade

Container não inicia?
  docker-compose logs
  docker-compose down && docker-compose build --no-cache && docker-compose up
